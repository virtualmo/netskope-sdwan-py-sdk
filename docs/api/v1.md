# Legacy v1 Module

The sections below summarize the GET methods backed by the uploaded v1 OpenAPI file.

The v1 spec uses camelCase wire parameter names such as `afterCursor` and `childTenantId`. In Python, the SDK exposes snake_case equivalents such as `after_cursor` and `child_tenant_id`.

## v1.edges.list

Returns v1 edges.

### Parameters
- `after_cursor` (`afterCursor`): optional cursor for records after the current page
- `before_cursor` (`beforeCursor`): optional cursor for records before the current page
- `max_items` (`maxItems`): optional maximum number of results
- `child_tenant_id` (`childTenantId`): optional child tenant selector
- `get_templates` (`getTemplates`): optional flag to include template records

### Response notes
- top-level response includes `data`, `startCursor`, `endCursor`, `firstPage`, and `lastPage`
- key edge fields include `id`, `description`, `model`, `role`, `interfaces`, `activated`, `assignedPolicy`, `dateCreated`, and `dateModified`
- edge `model` and `role` values are listed in [reference-values.md](../reference-values.md)

- the payload is a time-series object keyed by the selected metric
