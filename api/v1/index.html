    Legacy v1 Module¶

    The sections below summarize the GET methods backed by the uploaded v1 OpenAPI file.

    The v1 spec uses camelCase wire parameter names such as afterCursor and childTenantId. In Python, the SDK exposes snake_case equivalents such as after_cursor and child_tenant_id.

    v1.edges.list¶

    Returns v1 edges.

    Parameters¶

    • after_cursor (afterCursor): optional cursor for records after the current page
    • before_cursor (beforeCursor): optional cursor for records before the current page
    • max_items (maxItems): optional maximum number of results
    • child_tenant_id (childTenantId): optional child tenant selector
    • get_templates (getTemplates): optional flag to include template records

    Response notes¶

    • top-level response includes data, startCursor, endCursor, firstPage, and lastPage
    • key edge fields include id, description, model, role, interfaces, activated, assignedPolicy, dateCreated, and dateModified
    • edge model and role values are listed in reference-values.md

    v1.edges.get¶

    Returns one v1 edge by ID.

    Parameters¶

    • edge_id (edgeId): required edge ID
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • key fields include id, description, model, role, interfaces, activated, assignedPolicy, dateCreated, and dateModified

    v1.edges.list_interfaces¶

    Returns interfaces for an edge.

    Parameters¶

    • edge_id (edgeId): required edge ID
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • response is an array of interface objects
    • key fields include name, mode, type, addresses, macAddr, enableNat, isDisabled, and lteProps

    v1.edges.get_interface¶

    Returns one edge interface by name.

    Parameters¶

    • edge_id (edgeId): required edge ID
    • interface_name (interfaceName): required interface name
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • key fields include name, mode, type, addresses, macAddr, enableNat, isDisabled, and lteProps

    v1.edges.list_gateway_interfaces¶

    Returns gateway interfaces for a v1 gateway/edge ID.

    Parameters¶

    • edge_id (edgeId): required gateway ID in the v1 path shape
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • response is an array of interface objects
    • key fields include name, mode, type, addresses, macAddr, enableNat, isDisabled, and lteProps

    v1.edges.get_gateway_interface¶

    Returns one gateway interface by name.

    Parameters¶

    • edge_id (edgeId): required gateway ID in the v1 path shape
    • interface_name (interfaceName): required interface name
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • key fields include name, mode, type, addresses, macAddr, enableNat, isDisabled, and lteProps

    v1.monitoring.get_device_flows_totals¶

    Returns device flow totals for a gateway and time range.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector
    • start_datetime (startDatetime): required ISO 8601 UTC start timestamp
    • end_datetime (endDatetime): required ISO 8601 UTC end timestamp
    • ip: required device IP selector

    Response notes¶

    • response uses a top-level data array
    • key item fields include dest_ip, dest_port, dest_urls, avg_app_latency, concat_app_ids, concat_app_displaynames, concat_policies, and concat_peers

    v1.monitoring.get_devices_totals¶

    Returns device totals for a gateway and time range.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector
    • start_datetime (startDatetime): required ISO 8601 UTC start timestamp
    • end_datetime (endDatetime): required ISO 8601 UTC end timestamp

    Response notes¶

    • response uses a top-level data array
    • key item fields include name, mac_addr, ipv4, vnat_ips, flows_number, rx_bytes, tx_bytes, rx_packets, and tx_packets

    v1.monitoring.get_interfaces_latest¶

    Returns the latest interface overview for a gateway.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • response uses a top-level data array
    • key item fields include public_ip, cidr_ip, isp, type, state, bytes_rx, bytes_tx, packets_rx, packets_tx, cellular_stats, and wifi_stats

    v1.monitoring.get_paths_latest¶

    Returns the latest path overview for a gateway.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • response uses a top-level data array
    • key item fields include link, local_ip, overlay_ip, overlay_ip_name, latency, jitter, loss_rx, loss_tx, bytes_rx, bytes_tx, packets_rx, and packets_tx

    v1.monitoring.get_routes_latest¶

    Returns the latest route overview for a gateway.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector

    Response notes¶

    • response uses a top-level data array
    • key item fields include gateway_ip, gateway_name, interface_name, metric, netmask, and route_type

    v1.monitoring.get_system_load¶

    Returns system load data for a gateway and time range.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector
    • start_datetime (startDatetime): required ISO 8601 UTC start timestamp
    • end_datetime (endDatetime): required ISO 8601 UTC end timestamp
    • time_slots (timeSlots): optional number of buckets to return

    Response notes¶

    • response uses a top-level data field
    • the payload is a time-series object defined by the v1 monitoring schema

    v1.monitoring.get_paths_links¶

    Returns WAN path-link time-series data for a gateway, metric, and time range.

    Parameters¶

    • gateway_id (gatewayId): required gateway ID
    • child_tenant_id (childTenantId): optional child tenant selector
    • start_datetime (startDatetime): required ISO 8601 UTC start timestamp
    • end_datetime (endDatetime): required ISO 8601 UTC end timestamp
    • metric: required WAN metric. Valid values are listed in reference-values.md
    • time_slots (timeSlots): optional number of buckets to return

    Response notes¶

    • response uses a top-level data field
    • the payload is a time-series object keyed by the selected metric

    v1.monitoring.get_paths_links_totals¶

    Returns WAN path-link summary data for a gateway and time range.

    Parameters¶

    • child_tenant_id (childTenantId): optional child tenant selector
    • start_datetime (startDatetime): required ISO 8601 UTC start timestamp
    • end_datetime (endDatetime): required ISO 8601 UTC end timestamp

    Response notes¶

    • response uses a top-level data array
    • key item fields include link, ip, isp, avg_latency, avg_jitter, avg_loss, avg_rx_throughput, avg_tx_throughput, max_latency, and max_jitter

    v1.users.get_groups¶

    Returns groups for a user.

    Parameters¶

    • user_id (userId): required user ID
    • after_cursor (afterCursor): optional cursor for records after the current page
    • before_cursor (beforeCursor): optional cursor for records before the current page
    • max_items (maxItems): optional maximum number of results

    Response notes¶

    • top-level response includes data, startCursor, endCursor, firstPage, and lastPage
    • key item fields include id, name, templateId, autoProvision, dateCreated, and dateModified
